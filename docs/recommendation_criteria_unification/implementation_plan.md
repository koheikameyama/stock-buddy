# おすすめと「気になる」の判断基準の統一および安全性ルールの緩和

おすすめ銘柄として提案されたものが、「気になる」に追加した際のAI分析で即座に「見送り（stay）」判定になってしまう問題を解消し、全体的な安全性ルールの閾値を緩和します。

## 変更内容

### 共通ルール (`lib/stock-safety-rules.ts`, `lib/constants.ts`)

全体的な安全性ルールの閾値を緩和し、過度に保守的な判定を抑制します。

- 週間下落率の閾値を緩和:
  - 短期: -7% -> -10%
  - 中期: -10% -> -15%
  - 長期: -15% -> -20%
- 高ボラティリティの判定閾値を緩和 (50% -> 60%)

### 購入推奨コアロジック (`lib/purchase-recommendation-core.ts`)

- 当日のおすすめ銘柄（`UserDailyRecommendation`）については、モメンタムやボラティリティによる強制的な「様子見/見送り」判定をスキップするロジックを追加。
- 「見送り推奨（avoid）」のAI判定基準を厳格化し、致命的な問題がない限りは「様子見（stay）」を選択するようプロンプトを調整。

### おすすめ銘柄生成 (`app/api/recommendations/generate-daily/route.ts`)

- 以前追加した安全性フィルター（候補からの除外）を削除し、AIの判断を優先するように戻す。
