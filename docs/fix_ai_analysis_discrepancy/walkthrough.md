# 修正内容の確認 (Walkthrough) - AI分析の判断不一致の修正

## 修正のポイント

ウォッチリスト分析（購入判断）が、ポートフォリオ分析（売買判断）と同等の厳しいテクニカル評価基準を持つように修正しました。これにより、同じ悪条件の銘柄に対して「売るべきなのに買う」という矛盾したアドバイスが出るのを防ぎます。

### 1. テクニカル指標による強制補正 (Hard Override) の導入

`lib/purchase-recommendation-core.ts` に、ポートフォリオ分析と同じロジックを用いた強制補正を導入しました。

- **強い売りシグナル時のブレーキ**: ローソク足パターン、RSI、MACDを総合したスコアが「強い売り」を示している場合、AIの回答が "buy" であっても強制的に "stay"（様子見）に書き換えます。
- **一貫性の確保**: 今回の問題銘柄（ブライトパス・バイオ）のように、テクニカルが崩れている銘柄に対して、ポートフォリオ側とウォッチリスト側で一致した慎重なアドバイスが出るようになりました。

### 2. プロンプトの優先順位の調整

AIへの指示を強化し、提供された予測データ（今週は上昇見込みなど）と、目の前のリアルタイムなテクニカルシグナルが矛盾する場合、**安全性を優先して「様子見」を選ぶ**ように指示しました。

### 3. 分析コンテキストの拡充

ウォッチリスト側でも、ポートフォリオ側と同等の詳細な分析情報をAIに提供するようにしました。

- **出来高分析 (Volume Analysis)**: 「本物の売り圧力（分配売り）」か「一時的な調整」かを判定。
- **相対強度 (Relative Strength)**: 市場全体の下落か、銘柄固有の弱さかを判定。

## 検証結果

- **不一致の解消**: ブライトパス・バイオ（4594.T）のようにテクニカルが急激に悪化している銘柄において、ポートフォリオ側の「売却検討」とウォッチリスト側の「様子見（強い売りシグナルによる強制補正）」が一貫して表示されることを確認しました。
- **シンタックスエラーの解消**: 修正に伴い発生したインポートエラーや変数の重複定義をすべて修正し、ビルドが通る状態であることを確認しました。
