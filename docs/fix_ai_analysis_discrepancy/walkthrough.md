# 修正内容の確認 (Walkthrough) - AI分析の判断不一致の修正

## 修正のポイント

ウォッチリスト分析（購入判断）が、ポートフォリオ分析（売買判断）と同等の厳しいテクニカル評価基準を持つように修正しました。これにより、同じ悪条件の銘柄に対して「売るべきなのに買う」という矛盾したアドバイスが出るのを防ぎます。

### 1. 分析コンテキストの拡張

- **出来高分析（Volume Analysis）の導入**: 下落が「出来高を伴う本物（分配売り）」か「一時的な調整（出来高なし）」かをAIが判断できる情報を提供。
- **相対強度（Relative Strength）の導入**: 市場全体やセクター平均と比較して、その銘柄が「地合いで下げているだけ」か「個別に弱い」かをAIに明示。

### 2. プロンプトの刷新

- **「落ちるナイフ」回避ルールの明文化**: 強い売りシグナル（大陰線、出来高を伴う下落、市場対比の弱さ）がある場合は、成長性が高くても "buy" を出さず "stay"（様子見）とするよう指示を強化。
- **評価基準の同質化**: ポートフォリオ分析で使用している評価ロジック（テクニカル・モメンタム判断）をウォッチリスト分析プロンプトにも移植。

### 3. リスク管理の強化

- 短期的なテクニカルの崩れがある際のアドバイスがより具体的になり、初心者ユーザーが高値掴みやナンピン（下落中の買い増し）を避けるための指導をAIが行うようにしました。
