# 購入後分析シミュレーションの改善と不具合修正

ウォッチリスト銘柄の詳細ページに、「購入後分析シミュレーション」機能を使いやすく実装・改善し、最終的なUI調整も完了しました。

## 実施内容

### 1. UI配置と操作性の向上

- **ボタン配置**: 「AI購入判断」セクションの右上に「購入後分析シミュレーション」ボタンを配置。
- **モーダル化**: 結果をモーダルで表示。閉じるボタン（✕）を右上に固定し、コンテンツをスクロールしても常に表示されるようにしました。
- **不要な要素の削除**: シミュレーションモードでは不要な「再分析（更新）」ボタンを非表示にしました。

### 2. 自動分析と演出の最適化

- **自動開始**: モーダルを開いた瞬間に分析を自動開始（UXの向上）。
- **メッセージ**: 生成中のメッセージを「購入後分析を生成中...」にカスタマイズ。
- **表示バグ修正**: 生成開始時にスケルトン表示が残る不具合を解消し、即座に生成中メッセージが出るようにしました。

### 3. データとAPIの修正

- **APIエラー解消**: `405 Method Not Allowed` エラーを解消。
- **予測テキストの表示**: フィールド名の不一致を解消し、短期・中期・長期の予測解説テキストが正しく表示されるようにしました。

## 検証結果

- [x] シミュレーション中に「再分析（更新）」ボタンが表示されない。
- [x] 分析完了後、予測テキストが正しく表示される。
- [x] コンソールにエラーが出ていない。
